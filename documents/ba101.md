# Bandott 4K STB / Foxconn BA101

Device: Bandott 4K STB / Foxconn BA101

SoC: Amlogic S905D

Memory/storage: 2 GB RAM, Samsung 8 GB eMMC (`8GME4R`, KLM8G1GEME class)

Wireless module: Qualcomm Atheros QCNFA324

## Boot status

This device was validated booting Armbian from USB storage. eMMC installation is not recommended.

Use the BA101 DTB:

```text
FDT=/dtb/amlogic/meson-gxl-s905d-ba101.dtb
```

The validated setup used kernel `6.6.137-ophub` with the BA101 DTB copied to `/boot/dtb/amlogic/`.

## Ethernet

The internal Ethernet port is 100M, not Gigabit. The working DTB config uses RMII with the Meson GXL internal PHY.

Validated result:

```text
Speed: 100Mb/s
Duplex: Full
PHYAD: 8
Transceiver: internal
Link detected: yes
```

Relevant kernel log:

```text
PHY [mdio_mux-0.1:08] driver [Meson GXL Internal PHY]
configuring for phy/rmii link mode
Link is Up - 100Mbps/Full
```

## Wi-Fi

Wi-Fi works with the in-kernel `ath10k_sdio` driver. The module is detected as QCA6174 SDIO.

Validated kernel log:

```text
ath10k_sdio mmc0:0001:1: qca6174 hw3.2 sdio target 0x05030000
firmware ver WLAN.RMH.4.4.1-00174
```

Network scan was successful with `wlan0`.

## Bluetooth

Bluetooth is not validated yet. The current DTB lineage exposes the Bluetooth UART as a Broadcom-compatible device, but BA101 hardware uses the QCNFA324/QCA6174 family. BlueZ can start, but no usable default controller is available with that Broadcom DTB description.

Do not enable experimental QCA Bluetooth DTB changes by default until the UART GPIO, clock, and compatible string are verified.

## eMMC

The internal eMMC is readable, but installing Armbian/OpenWrt to BA101 eMMC is not recommended.

Observed eMMC:

```text
/dev/mmcblk2       7.28 GiB
/dev/mmcblk2boot0  4 MiB
/dev/mmcblk2boot1  4 MiB
/dev/mmcblk2rpmb   512 KiB
```

Health data from `mmc extcsd read /dev/mmcblk2`:

```text
EXT_CSD_PRE_EOL_INFO = 0x01
EXT_CSD_DEVICE_LIFE_TIME_EST_TYP_A = 0x0b
EXT_CSD_DEVICE_LIFE_TIME_EST_TYP_B = 0x0b
```

`LIFE_TIME_EST_TYP_A/B=0x0b` means the estimated lifetime is exceeded. The stock layout also uses Amlogic MPT rather than a standard Linux partition table, and community reports indicate that writing Linux images to BA101 eMMC can cause black-screen boot failure.

Recommended operation:

- Boot Armbian from USB.
- Keep eMMC untouched except for read-only inspection and backups.
- Treat eMMC use as risky and non-recoverable without a known-good stock restore path.
